## Konventionen

- Checkbox = trackbarer Umsetzungsstatus
- Jeder Ticket-Block beschreibt Ziel, Scope, Hauptbereiche und Abhaengigkeiten
- Keine Schattenlogik: neue Tickets muessen bestehende zentrale Wahrheiten wiederverwenden
- Bei inhaltlichen Aenderungen zuerst die zugehoerige Epic in `docs/epics/` und dann den Plan anpassen
