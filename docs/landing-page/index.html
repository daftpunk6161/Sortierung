Core Features

Everything you need to manage, verify, deduplicate, and convert your ROM collection.

🎯

Region-Based Deduplication

Deterministic scoring engine with region priority, format scoring, and version scoring. Keep one best ROM per game — same result every time.

📋

DAT Verification

Multi-source DAT matching (Redump, No-Intro, TOSEC, Non-Redump). Pre-Detection DAT Lookup, Cross-Console Hash Lookup, 6 Family Strategies.

🔄

Format Conversion

Multi-step conversion pipeline with source backup, tool hash verification, and output validation. Supports chdman, 7-Zip, NKit, and more.

🔒

Signed Audit Trail

SHA256-signed audit logs with full rollback capability. Every file operation is tracked. Undo any run — unique to Romulus.

🩺

Collection Health Monitor

Continuous integrity checks, bit-rot detection, integrity scoring, and alerts. Know your collection's health at a glance.

️

Three Entry Points

WPF GUI for interactive use, CLI for automation and CI, REST API with SSE for integration. Same deterministic engine behind all three.

🐋

Docker Ready

Run the CLI and API headless in Docker. Perfect for NAS, Steam Deck, or CI pipelines.

Why Romulus?

What sets it apart from every other ROM manager.

🛡️

No Data Loss — Ever

Default behavior is Move to Trash + Audit + Undo. Source files are never deleted before verification succeeds. Partial outputs are cleaned up on failure.

⚖️

Deterministic Pipeline

Preview shows exactly what Execute will do. Reports count the same decisions. GUI, CLI, and API produce identical results for the same input.

🔑

Per-Console Policies

163 consoles with differentiated rules: conversion policies, DAT strategies, scoring profiles, region priorities. Not "one size fits all".

🧪

Battle-Tested

7,100+ tests including unit, integration, regression, negative/edge cases. Security hardened: path traversal, zip-slip, CSV injection, tool hash verification.

🎛️

Built-in Profiles

Start with a preset (default, retro-purist, space-saver, quick-scan) or create custom profiles for your exact workflow.

Quick Start

Three ways to run Romulus.

🖥️

GUI (Windows)

Download the release, extract, run Romulus.Wpf.exe. Add your ROM roots, pick a profile, preview, and execute.

⌨️

CLI

romulus run --roots "D:\ROMs" --profile default --execute
Headless, CI-friendly, scriptable.

🌐

REST API + Docker

docker compose up with the provided docker-compose.headless.yml. API runs on port 5000 with SSE for real-time progress.
