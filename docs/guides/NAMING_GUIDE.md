# Naming-Guide

> **Status:** ❌ ARCHIVIERT (2026-03-11)
>
> Dieses Dokument beschrieb PowerShell-spezifische Benennungskonventionen (Verb-Noun, `$camelCase`, `.ps1`-Modulnamen).
> Die PowerShell-Version wurde vollständig nach C# .NET 10 migriert.
>
> **Aktuelle Quelle für C#-Naming-Konventionen:**
> - `.github/copilot-instructions.md` → Abschnitt "Schlüssel-Konventionen / Code-Struktur"
> - `.claude/rules/cleanup.instructions.md` → Abschnitt "Grundprinzipien"
>
> **C#-Kurzfassung:**
> - PascalCase für Methoden, Properties, Klassen
> - camelCase für lokale Variablen und Parameter
> - Dateien: `<Klasse>.cs`, Tests: `<Klasse>Tests.cs`
> - JSON-Daten: kebab-case (`consoles.json`, `tool-hashes.json`)
> - XAML: PascalCase (`MainWindow.xaml`)
> - Code-Kommentare, Log-Nachrichten, Commit-Messages: Englisch
> - UI-Texte: Deutsch via `data/i18n/de.json`
> - Dokumentation: Deutsch
>
> PowerShell-Originalversion: `archive/powershell/`
